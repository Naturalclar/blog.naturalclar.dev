9:T1565,<p>React Native で開発していて、新しい Component を作成するとき、どういうファイルを作成するでしょう。
React を Import するので、当然、全てのファイルには</p>
<pre><code class="language-jsx">import React from 'react'
</code></pre>
<p>と入ります。</p>
<p>基本的には<code>View</code>で Component を囲むし、何かしらの Style もつけるため、<code>StyleSheet</code>も必要となります。</p>
<pre><code class="language-jsx">import { View, StyleSheet } from 'react-native'
</code></pre>
<p>class component か、function component にするか、選択の余地があるが、最近は hooks があるので、これから書き始める Component は全て function component でいいでしょう。
何かしら Props が入る可能性は高いので、Props の型も定義しておきます。</p>
<pre><code class="language-jsx">
type Props = {}

const Foo = ({}:Props) = {
  return (
    &#x3C;View>&#x3C;/View>
  )
}
</code></pre>
<p>これら一連の動作は、ほとんど全ての Component ファイルで行います。
新しいファイルを作るたびにそれらを入力する hotkey が設定されていれば楽かもしれません。
他のファイルからその部分をコピペしてきても、消す手間や命名を変える手間はありますが、まあ許容できるかもしれません（個人的にはコピペするのすら面倒なので嫌です）。</p>
<p>もし、開発に storybook を使っていたり、snapshot test を実施していたりしたらどうなるでしょうか。
作成する Component ファイルの作成が必要な他、<code>.story.tsx</code>ファイルや、<code>.test.ts</code>ファイルを新たに作成する必要があります。
Component を作成するたびに上記作業を行うのはとても手間になります。</p>
<h2>scaffdog の導入</h2>
<p>Component を大量に作成するときに、毎回規定のファイルを作成するのが面倒なので、
自分は <a href="https://github.com/cats-oss/scaffdog">scaffdog</a> というツールを使用しています。</p>
<p>Scaffdog は markdown 式に Template を作成出来る scaffolding ツールです。
例えば、以下のような markdown のファイルを作成します。</p>
<pre><code class="language-md">---
name: 'atom'
description: 'Atom component template'
message: 'Please enter the name of component to be created'
root: './src/atoms'
output: '**/*'
ignore: []
---

# `{{ input }}.tsx`

```jsx
import React from 'react'
import { View, StyleSheet } from 'react-native'

const styles = StyleSheet.create({
  container: {
    flex: 1
  }
})

type Props = {}

export const {{ input }} : React.FC&#x3C;Props> = ({}: Props):JSX.Element => (
&#x3C;View style={styles.container}>&#x3C;/View>
)


```

# `{{ input }}.story.tsx`

```jsx
import React from 'react';
import { storiesOf } from '@storybook/react';
import {{ input }} from './{{input}}';
import { StorybookContainer } from "../utils";

storiesOf("atoms", module)
  .addDecorator(StorybookContainer)
  .add("{{input}}", () => (
  &#x3C;{{ input }} />
));
```
</code></pre>
<p>この markdown と scaffdog を使って、Foo という Component を作成すると、以下のファイルが生成されます。</p>
<p><code>Foo.tsx</code></p>
<pre><code class="language-jsx">import React from 'react'
import { View, StyleSheet } from 'react-native'

const styles = StyleSheet.create({
  container: {
    flex: 1,
  },
})

type Props = {}

export const Foo: React.FC&#x3C;Props> = ({  }: Props): JSX.Element => (
  &#x3C;View style={styles.container} />
)
</code></pre>
<p><code>Foo.story.tsx</code></p>
<pre><code class="language-jsx">import React from 'react'
import { storiesOf } from '@storybook/react'
import Foo from './Foo'
import { StorybookContainer } from '../utils'

storiesOf('atoms', module)
  .addDecorator(StorybookContainer)
  .add('Foo', () => &#x3C;Foo />)
</code></pre>
<p>このように、新しく Component を作成するときに毎回書き出すことを自動的に用意してくれるので、Component 作成が効率よく行えます。</p>
<h2>使い方</h2>
<p>scaffdog の使い方はとてもシンプルです。</p>
<p>まず、プロジェクトに <code>.scaffdog</code> というフォルダを作成します。</p>
<p>そして、そのフォルダの中に、先程のような template が書かれた markdown を入れます。</p>
<p>そして、scaffdog を呼び出します。</p>
<p>npx 経由で呼び出す場合、</p>
<pre><code>npx scaffdog generate
</code></pre>
<p>を入力し、作成したい Component の名前を入力します。</p>
<p>自分は、scaffdog をプロジェクトに入れて、<code>yarn new</code> で新しく Component を作成出来るようにしています。</p>
<h2>scaffdog の他の活用法</h2>
<p>scaffdog は、Component 以外にも、Gatsby で新しく Blog 記事を作成するときや、mdx-deck で新しく Slide を作成するときなど、様々な場面で使用することができます。</p>
<p>自分が使っているものは、以下のリポジトリの<code>.scaffdog</code> フォルダに入っているので、ぜひ確認してください。</p>
