9:T14e5,<p>最近、React Native の最新バージョン、0.62 がリリースされました。
React Native 0.62 の新機能については<a href="https://blog.naturalclar.dev/whats-new-in-react-native-0.62/">この記事</a>にて記載しています。</p>
<p>そして、次のバージョンの Release Candidate である 0.63-rc.0 も新しくリリースされました。
この記事では、次のバージョン、React Native 0.63 で予定されている新機能についてまとめます。</p>
<h2>React Native の Release について</h2>
<p>まず、React Native のリリースプロセスについて、少し説明します。
React Native は facebook が開発しているフレームワークですが、リリースプロセスは facebook 社外のコミュニティである React Native Community が行っています。
そのリリースプロセスや、各バージョンでリリースされる機能については、<a href="https://github.com/react-native-community/releases">@react-native-community/releases</a> のリポジトリに記載されています。</p>
<p>0.63 で予定されている機能は<a href="https://github.com/react-native-community/releases/issues/186">この issue</a> にて議論されています。</p>
<h2>React Native 0.63 新機能</h2>
<p>次のバージョンで予定されていることは以下の通りです。</p>
<ul>
<li>LogBox</li>
<li>Pressable</li>
<li>PlatformColor API</li>
<li>DynamicColorIOS API</li>
</ul>
<h3>LogBox</h3>
<p><img src="./logbox.png" alt="logbox"></p>
<p>LogBox は React Native 0.62 で試験的に導入された、新しいエラー画面です。
0.62 ではオプトインでしたが、次のバージョンからは、デフォルトで入るようになります。</p>
<p>今まで使われていた YellowBox は deprecated となり、LogBox API に置き換わります。</p>
<h3>Pressable</h3>
<p>Pressable は TouchableOpacity や TouchableHighlight のような新しい　onPress を持つ Component となります。</p>
<p>Pressable は「押された状態」を持っています。子 Component に <code>pressed</code> の状態をもたせ、押された状態と押されていない状態で見た目を変更するなどが行えるようになります。</p>
<pre><code class="language-jsx">import { View, Pressable, Text } from 'react-native'

const PressableExample = () => {
  return (
    &#x3C;View>
      &#x3C;Pressable
        onPress={() => {
          console.log('Event fired!')
        }}
      >
        {({ pressed }) => &#x3C;Text>{pressed ? 'Pressed!' : 'Press Me'}&#x3C;/Text>}
      &#x3C;/Pressable>
    &#x3C;/View>
  )
}
</code></pre>
<h3>PlatformColor</h3>
<p>PlatformColor は端末で設定されている基本色を取得するのに使用されます。</p>
<p>iOS なら UI Standard Colors や UI Element Colors のような <a href="https://developer.apple.com/documentation/uikit/uicolor">UI Colors</a>, Android は [R.attrs](- <a href="https://developer.android.com/reference/android/R.attr">Android R.attr</a> に設定されている値が取得できます。</p>
<p>それぞれ、DarkMode と LightColor の色が用意されているので、この API を使うだけで、自前で LightMode と DarkMode の色を設定しなくても Dark Mode の対応が行えるようになっています。</p>
<pre><code class="language-jsx">import { View, Text, StyleSheet, Platform, PlatformColor } from 'react-native'

const PlatformColorExample = () => {
  return (
    &#x3C;View style={{ backgroundColor: labelColor }}>
      &#x3C;Text style={styles.label}>Hello World!&#x3C;/Text>
    &#x3C;/View>
  )
}

const styles = StyleSheet.create({
  label: {
    ...Platform.select({
      ios: { color: PlatformColor('labelColor') },
      android: { color: PlatformColor('?attr/colorControlNormal') },
      default: { color: 'black' },
    }),
  },
})
</code></pre>
<h3>DynamicColorIOS</h3>
<p>DynamicColorIOS は Appearance に合わせた色を取得できる iOS専用の新しい API です。</p>
<p>light mode, dark mode でそれぞれ使用する色を設定することができます。</p>
<pre><code class="language-jsx">const headerTextColor = DynamicColorIOS({
  dark: 'white',
  light: 'black',
})
</code></pre>
<p>0.62 にて追加された <code>Appearance</code> API や、今回追加される <code>PlatformColor</code> や <code>DynamicColorIOS</code> API の追加によって、Dark Mode 対応はよりやりやすくなるでしょう。</p>
<h2>その他の変更点</h2>
<h3>iOS 9 のサポート切り</h3>
<p>iOS 9 のサポートが切られます。
iPhone 4 以前の端末の対応が必要な場合、アップデートするのは控える必要があるかも。</p>
<h3>Node.js 8 のサポート切り</h3>
<p>先日、EOL を迎えた Node.js 8 のサポートが切られます。
React Native の開発には Node.js 10以降のバージョンを使う必要があります。</p>
<h3>Hermes 0.5.0</h3>
<p><a href="https://github.com/facebook/hermes/releases">アップデートされた Hermes 0.5.0</a> が入ります。
新しい version では、hermes のサイズが 140MB から 18MB に削減されます。</p>
<h2>リリース予定日</h2>
<p>次のバージョンのリリースは5月中旬を予定されています。</p>
