9:Ta43,<p>React Native で typescript の paths の alias を適用するにはいくつか方法がありますが、この記事では追加のライブラリを必要としない方法を紹介します。</p>
<h3>Paths Alias とは</h3>
<p>typescript における paths の設定は適用すると特定のディレクトリからのインポートを相対的に行わずにかけるようになります。</p>
<p>例えば、<code>Button</code> というコンポーネントが <code>src/components/Button.tsx</code> にあるとします。</p>
<p>paths alias を適用する前のコード</p>
<pre><code class="language-js">// src/screens/LoginScreen.tsx

import * as React from 'react';
import { View } from 'react-native';
import { Button } from '../components/Button';

export const LoginScreen = () => {
  ...
}
</code></pre>
<p>paths alias を適用した後のコード</p>
<pre><code class="language-js">// src/screens/LoginScreen.tsx

import * as React from 'react';
import { View } from 'react-native';
import { Button } from 'app/components/Button';

export const LoginScreen = () => {
  ...
}
</code></pre>
<p>上記のように、<code>../components/Button</code> から <code>app/components/Button</code> に変更されます。
今回のケースではそこまで恩恵を感じないかもしれませんが、LoginScreenをより深い階層のディレクトリに移動する場合等にコードの変更を行わずにそのまま利用することができるようになります。</p>
<p>paths alias を適用する場合、<code>tsconfig.json</code> では以下のように設定します。</p>
<pre><code class="language-json">{
  "compilerOptions": {
    "baseUrl": "./",
    "paths": {
      "app/*": ["src/*"]
    }
  }
}
</code></pre>
<p>ただ、この設定だけだとVSCode等のエディタ上ではエラーが表示されませんが、実際にReact Nativeのアプリを起動した時にはエラーになります。</p>
<p>アプリを起動した時に設定が反映されるようにする方法はいくつかありますがここでは <code>metro.config.js</code> を変更することで解決させます。</p>
<pre><code class="language-js">const { getDefaultConfig, mergeConfig } = require('@react-native/metro-config')
const path = require('node:path')

/**
 * Metro configuration
 * https://reactnative.dev/docs/metro
 *
 * @type {import('metro-config').MetroConfig}
 */
const config = {
  resolver: {
    extraNodeModules: {
      app: path.resolve(__dirname, 'src'),
    },
  },
}

module.exports = mergeConfig(getDefaultConfig(__dirname), config)
</code></pre>
